## 📝 雨课堂核心作业大题保姆级解析

[00雨课堂作业.docx](3%20计算机系统结构/00雨课堂作业.docx) 是岳承君老师平时作业的主要发布渠道，其中的 5 大题目不仅是期末试卷的直接来源，也深刻揭示了系统结构的核心理论：

### 大题 1：系统结构、组成与实现的定义与关系（以 Intel x86 为例）
* **系统结构（ISA）**：程序员看到的计算机属性（如指令集、数据类型、寻址方式），是软硬件契约（“What to do?”）。如 x86-64 指令集。
* **计算机组成（微架构）**：系统结构的逻辑实现（“How to do it?”）。如 Conroe 微架构（双核、短流水线）和 Cypress Cove 微架构（8核、乱序执行、三级 Cache 架构）。
* **计算机实现**：组成的物理体现与制造工艺（“With what to build it?”）。如 65nm 与 14nm 制程、时钟频率、封装设计等。
* **相互关系**：
  * 系统结构是稳定的，组成是演进最活跃的，实现是物理支撑。
  * 上层（系统结构）决定下层（组成）的基本功能；下层（实现）的物理瓶颈（如“功耗墙”、“温度墙”）反过来制约上层的微架构设计。

### 大题 2：CISC 与 RISC 架构特点与融合
* **CISC（复杂指令集）**：指令系统庞大复杂，指令长度可变，寻址方式多。
  * **不足**：执行速度慢，难以优化编译，指令使用频度极低（20% 的简单指令占了 80% 的运行时间）。
* **RISC（精简指令集）**：指令简单、等长，多单周期指令，仅用 Load/Store 访问内存。
  * **不足**：导致编译出的源程序代码变长，指令信息流量大，对浮点数及虚存支持不够。
* **融合发展**：现代高性能处理器（如 Intel Core / AMD Ryzen）通常采用 **CISC 的外壳（兼容 x86 指令），内部译码为类似 RISC 的微操作（uops）执行**，融合了 CISC 的高兼容性与 RISC 的硬件高效性。

### 大题 3：多级中断处理屏蔽字与执行顺序计算
* **问题背景**：设某机有 4 级中断，响应优先级按硬件查询链为 $1 \to 2 \to 3 \to 4$。如果希望实际的中断处理完毕次序为 $1 \to 3 \to 4 \to 2$，各级屏蔽位（1为屏蔽，0为开放）如何设置？
* **屏蔽字推导原则**：
  * 中断屏蔽位由该中断源**服务程序**所对应的屏蔽寄存器决定。
  * 处于执行状态下的中断服务程序应当屏蔽处理优先级低于自身（包括同级）的中断，开放处理优先级高于自身的中断。
  * 实际处理完毕的顺序反映了处理优先级的真实高低：$1 > 3 > 4 > 2$。
* **各级屏蔽位（格式为 `[对应1级, 对应2级, 对应3级, 对应4级]`）**：
  * 1 级处理优先级最高：屏蔽所有级别 $\implies$ 屏蔽位为 `[1, 1, 1, 1]`。
  * 3 级处理优先级第二：仅对 1 级开放，屏蔽 2, 3, 4 级 $\implies$ 屏蔽位为 `[0, 1, 1, 1]`。
  * 4 级处理优先级第三：对 1, 3 级开放，屏蔽 2, 4 级 $\implies$ 屏蔽位为 `[0, 1, 0, 1]`。
  * 2 级处理优先级最低：对 1, 3, 4 级开放，屏蔽自己 $\implies$ 屏蔽位为 `[0, 1, 0, 0]`（注：屏蔽自己一般也设为1，即 `[0, 1, 0, 0]`）。

### 大题 4：Cache 容量与速度取舍决策
* **平均访问时间公式**：
  $$T_{avg} = H \times T_c + (1-H) \times T_m$$
  *(其中 $H$ 为命中率，$T_c$ 为 Cache 访问时间，$T_m$ 为主存访问时间)*
* **决策逻辑**：
  * 若当前命中率较低（例如 $<90\%$），说明主要瓶颈在于不命中时访问慢速主存的开销（即阻碍性能的是 $(1-H) \times T_m$ 部分）。此时**扩充 Cache 容量**能大幅提高命中率 $H$，获得最高的边际效益。
  * 若当前命中率已经极高（例如 $>95\%$），说明 $(1-H) \times T_m$ 已经微乎其微。此时增加容量不仅性价比极低，还会增大命中延迟。此时应干脆**购买更高速的 Cache 片子**降低 $T_c$，才能直接优化性能。

### 大题 5：向量处理机链接技术时钟节拍（拍数）计算（★★★★★ 期末大题必考）
* **已知条件**：CRAY-1 机器启动存储器、流水部件及寄存器打入各需 1 拍。“加法”延时为 6 拍（浮加 6 拍），“乘法”延时为 7 拍（浮乘 7 拍），“访存”延时为 6 拍。现有向量指令串（向量长度为 $N$）：
  1. $V_3 \leftarrow \text{存储器}$ （访存，延时为 6 拍）
  2. $V_2 \leftarrow V_0 + V_1$ （浮加，延时为 6 拍）
  3. $V_4 \leftarrow V_2 \times V_3$ （浮乘，延时为 7 拍，需要用到 $V_2$ 和 $V_3$ 的结果）
* **（1）串行执行所需拍数**：
  * 指令 1 耗时：$1(\text{启动}) + 6(\text{访存}) + 1(\text{存入}) + N - 1 = N + 7$ 拍。
  * 指令 2 耗时：$1(\text{启动}) + 6(\text{加法}) + 1(\text{存入}) + N - 1 = N + 7$ 拍。
  * 指令 3 耗时：$1(\text{启动}) + 7(\text{乘法}) + 1(\text{存入}) + N - 1 = N + 8$ 拍。
  * **总耗时**：$(N+7) + (N+7) + (N+8) = 3N + 22$ 拍。
* **（2）指令 1 与 2 并行，然后再串行执行指令 3**：
  * 前两条指令无数据相关，可并行执行。耗时为 $\max(N+7, N+7) = N+7$ 拍。
  * 两者执行完毕后，再串行启动指令 3，耗时 $N+8$ 拍。
  * **总耗时**：$(N+7) + (N+8) = 2N + 15$ 拍。
* **（3）采用链接技术执行**：
  * 指令 1 和 2 可以同时在第 0 拍启动。
  * 第 8 拍时，指令 1 和 2 的第一个结果分量分别到达寄存器 $V_3$ 和 $V_2$。
  * 链接器在第 8 拍直接把这两个分量送入乘法器启动指令 3。此时指令 3 的输入不必等所有向量分量读/加完。
  * 乘法器第一个分量输出需要：$1(\text{启动}) + 7(\text{乘法}) + 1(\text{存入}) = 9$ 拍。也就是说，在第 $8 + 9 = 17$ 拍，第一个分量写入 $V_4$。
  * 剩下的 $N-1$ 个分量以流水线方式每隔一拍输出一个，所以还要 $N-1$ 拍。
  * **总耗时**：$8(\text{链接起始拍}) + 9(\text{乘法延时}) + N - 1 = N + 16$ 拍。

## 🎯 期末核心计算大题必杀详解

### 题型 1：虚页地址变换与实地址计算（10分）
* **真题演示**：某虚拟存储器共 8 个页面，每页为 1024 个字，实际主存为 4096 个字（即 4 个实页，实页号 0~3）。地址映像表如下：
  | 虚页号 | 实页号 | 装入位 |
  | :---: | :---: | :---: |
  | 0 | 3 | 1 |
  | 1 | 1 | 1 |
  | 2 | 4 | 0 |
  | 3 | 3 | 0 |
  | 4 | 2 | 1 |
  | 5 | 2 | 0 |
  | 6 | 0 | 1 |
  | 7 | 1 | 0 |
* **（1）页面失效的虚页号**：装入位为 `0` 的项，即 **2, 3, 5, 7** 虚页。
* **（2）按以下虚地址计算主存实地址**：（公式：$p = A_V // 1024$，$d = A_V \% 1024$，若装入位为1，实地址 $A_R = \text{实页号} \times 1024 + d$）
  * **3700**：$3700 // 1024 = 3$，虚页 3 装入位为0 $\implies$ **页面失效（缺页中断）**。
  * **1000**：$1000 // 1024 = 0$, 页内地址 $1000$。虚页 0 映射至实页 3 $\implies$ 实地址 $= 3 \times 1024 + 1000 = 4072$。
  * **1026**：$1026 // 1024 = 1$, 页内地址 $2$。虚页 1 映射至实页 1 $\implies$ 实地址 $= 1 \times 1024 + 2 = 1026$。
  * **5000**：$5000 // 1024 = 4$, 页内地址 $904$。虚页 4 映射至实页 2 $\implies$ 实地址 $= 2 \times 1024 + 904 = 2952$。
  * **5**：$5 // 1024 = 0$, 页内地址 $5$。虚页 0 映射至实页 3 $\implies$ 实地址 $= 3 \times 1024 + 5 = 3077$。
  * **7004**：$7004 // 1024 = 6$, 页内地址 $860$。虚页 6 映射至实页 0 $\implies$ 实地址 $= 0 \times 1024 + 860 = 860$。

### 题型 2：Cache 组相联映像与 LRU 替换算法（10分）
* **真题演示**：主存有 8 个块（块 0~7），Cache 有 4 个块（块 0~3）。采用组相联映像，每组 2 块，采用近期最少使用（LRU）替换算法。
* **（1）映像规则描述与组划分**：
  * Cache 块数为 4，每组 2 块，共分为 2 组。
  * 组号 $g = \text{主存块号} \pmod 2$。
  * 组 0 对应 Cache 块 0 和 1，可映射主存块 0, 2, 4, 6。
  * 组 1 对应 Cache 块 2 和 3，可映射主存块 1, 3, 5, 7。
* **（2）主存块地址流状态流演变**：（访问序列：1、2、4、1、3、7、0、1、2、5、4、6）
  * 我们分别对组0和组1的缓存队列进行更新。队列左端为最久未访问（若满，则替换之），右端为最近访问（最新装入或命中的块）。
  
  | 访问块 | 对应组 | 组0状态队列（Cache块0,1） | 组1状态队列（Cache块2,3） | 结果 |
  | :---: | :---: | :---: | :---: | :---: |
  | **1** | 组 1 | `[ ]` | `[1]` | ❌ 未命中 |
  | **2** | 组 0 | `[2]` | `[1]` | ❌ 未命中 |
  | **4** | 组 0 | `[2, 4]` | `[1]` | ❌ 未命中 |
  | **1** | 组 1 | `[2, 4]` | `[1]` | 🟢 命中 |
  | **3** | 组 1 | `[2, 4]` | `[1, 3]` | ❌ 未命中 |
  | **7** | 组 1 | `[2, 4]` | `[3, 7]` *(替换最久未访问的1)* | ❌ 未命中 |
  | **0** | 组 0 | `[4, 0]` *(替换最久未访问的2)* | `[3, 7]` | ❌ 未命中 |
  | **1** | 组 1 | `[4, 0]` | `[7, 1]` *(替换最久未访问的3)* | ❌ 未命中 |
  | **2** | 组 0 | `[0, 2]` *(替换最久未访问的4)* | `[7, 1]` | ❌ 未命中 |
  | **5** | 组 1 | `[0, 2]` | `[1, 5]` *(替换最久未访问的7)* | ❌ 未命中 |
  | **4** | 组 0 | `[2, 4]` *(替换最久未访问的0)* | `[1, 5]` | ❌ 未命中 |
  | **6** | 组 0 | `[4, 6]` *(替换最久未访问的2)* | `[1, 5]` | ❌ 未命中 |
* **（3）命中率计算**：
  * 总访问次数：12 次。
  * 命中次数：1 次（第4次访问1）。
  * 命中率 $H = 1 / 12 \approx 8.33\%$。

2. **“早做创新学分转换规划”**：大学四年里参加的全国性学科竞赛、申请的软件著作权或发表的专利，在第七学期都可以去教务处申请实践环节学分转换。提早准备证明材料，不仅能拿到 95 分的高分绩点，还可以为你空出宝贵的复习考研和实习求职时间。
3. **收官之战，稳扎稳打**：大四没有太多繁杂课程的干扰，静下心来把最后一门专业硬课考好。预祝各位学弟学妹在秋招、保研或考研战场上顺利靠岸，开启人生新篇章！
